Understanding the Structure of a JavaScript Program

Welcome to your first look at the building blocks of JavaScript! Every program you write is made up of simple, structured pieces. Understanding these pieces‚Äîlike comments, statements, and data types‚Äîis the key to writing clear, functional code.

1. Program Structure: Statements and Comments

Statements (The Instructions)

A JavaScript statement is a complete instruction to the computer. Statements are typically separated by a semicolon (;), which acts like a period at the end of a sentence.

let message = 'Hello, World!';  // Statement 1: Declares a variable
console.log(message);             // Statement 2: Prints the variable's value

Comments (The Notes)

Comments are text the computer ignores, used only by humans to explain what the code does. This is vital for organization and clarity, especially for ADHD learners who benefit from breaking down complex logic.

// This is a single-line comment

// Print Hello, World! to the console
console.log('Hello, World!');
Try It: Open your browser's Developer Console (usually F12) and type console.log('Hello, World!');. This is the first step of debugging!

2. Variables and Constants

Variables are like labelled boxes where you can store information (data). In JavaScript, you use keywords like let and const to create them.

Variable Declaration (let)

Use let for values that you expect to change later in the program.

let message = 'Hello, World!';
// Later in the program, you can change it:
message = 'Goodbye, World!';

Constants (const)

Use const for values that should *never* change. This makes your code safer and easier to understand.

let age = 25;
const pi = 3.14; // Pi will always be 3.14 (a constant)

3. JavaScript Data Types

Every piece of data in JavaScript has a type. Understanding these types is crucial for making sure your program handles information correctly.

Type Description Code Snippet
Number Any numerical value (integers or decimals). 
let score = 100;
String A sequence of characters, always enclosed in quotes ('' or ""). 
let greeting = 'Hello, World!';
Boolean A logical value: either true or false. Used for decision-making. 
let isStudent = true;
Null Represents the intentional absence of any object value. (You set it to null). 
let emptyValue = null;
Undefined A variable that has been declared but has not been assigned a value. 
let notAssigned;
Object A complex data type used to store collections of data and more complex entities (like a real-world object). 
let person = {
 name: 'John',
 age: 30
};
Concept Tip (Object): Think of an Object as a filing cabinet. The person is the cabinet, and name and age are the labels on the drawers, each holding a specific piece of information.
