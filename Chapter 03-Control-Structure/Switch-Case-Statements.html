Switch-Case Statements

The switch-case statement is a powerful control flow tool used when you need to execute different code blocks based on the exact value of a single expression. It is often much cleaner and easier to read than a long chain of if...else if statements.

Analogy: Think of a vending machine. You put in your selection (switch(selection)). The machine checks the code you pressed (case 'A1':). When it finds a match, it dispenses the item (break; prevents it from dispensing all items).

The `switch-case` Structure: Day Example

The expression (day in this example) is evaluated once. The result is then compared with the values of each case label.

let day = 3;
let dayName;

switch (day) {
    case 1:
        dayName = 'Monday';
        break;
    case 2:
        dayName = 'Tuesday';
        break;
    case 3:
        dayName = 'Wednesday'; // Match! Code executes here.
        break; // Stops the switch and exits.
    case 4:
        dayName = 'Thursday';
        break;
    case 5:
        dayName = 'Friday';
        break;
    case 6:
        dayName = 'Saturday';
        break;
    case 7:
        dayName = 'Sunday';
        break;
    default:
        dayName = 'Invalid day';
}

// Outputs: Wednesday
console.log(dayName);

The Code Behind the Day Finder

This function uses the switch-case statement to translate the number input into the day name:

function checkDayName() {
    let day = Number(document.getElementById('day-number').value);
    let dayName;
    
    switch (day) {
        case 1: dayName = 'Monday'; break;
        case 2: dayName = 'Tuesday'; break;
        case 3: dayName = 'Wednesday'; break;
        case 4: dayName = 'Thursday'; break;
        case 5: dayName = 'Friday'; break;
        case 6: dayName = 'Saturday'; break;
        case 7: dayName = 'Sunday'; break;
        default: dayName = 'Invalid day number entered.';
    }
    
    document.getElementById('result-output').innerText = `Day ${day} is: ${dayName}`;
}

Key Takeaways

  • The switch-case is best for checking one variable against multiple specific values.
  • The break keyword is crucial: it prevents "fall-through" by exiting the switch block once a match is found. Without it, the code would execute the next cases too!
  • The default case is optional but recommended; it runs if none of the case values match the expression.
