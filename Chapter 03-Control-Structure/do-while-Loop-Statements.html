The `do-while` Loop Statement

The do-while loop is very similar to the standard while loop, with one crucial difference: it executes the code block first, and then checks the condition second. This guarantees that the loop body will always run at least one time, regardless of whether the condition is initially true or false.

Analogy: Think of an exit requirement for a game level. You start the level, you play (the do part). Only when you reach the end, do we check the score: WHILE the score is below the target, let the player try again. But the player always plays at least once.

The Structure of the `do-while` Loop

Notice that the condition is at the very end of the loop, following the while keyword, which requires a semicolon.

let number = 0; // 1. Initialization

do
{
    console.log('Number: ' + number);
    
    number++; // 2. The Updater runs here
} while (number < 5); // 3. Condition is checked at the end (note the semicolon!)

Key Parts Explained:

  • The do Block: The code inside the curly braces runs immediately.
  • The while Condition: The loop checks this condition after the do block has executed.
  • The Critical Difference: If you set let number = 10;, a standard while loop would skip entirely, but a do-while loop would still execute the body once (printing Number: 10) before exiting.

The Code We Are Running

This function executes the requested loop and displays the output:

function runDoWhileLoop() {
    // Change this number to 5 or 10 to see the "run once" effect!
    let number = 0; 
    let output = '';

    do {
        output += ('Number: ' + number + '\n');
        number++;
    } while (number < 5);

    document.getElementById('result-output').innerText = output;
}

Key Takeaways

  • The do-while loop is used when you need to run the code at least once. (e.g., getting input from a user, where you always prompt them first).
  • The code block runs before the condition check.
  • The while part of the statement requires a semicolon (;).
