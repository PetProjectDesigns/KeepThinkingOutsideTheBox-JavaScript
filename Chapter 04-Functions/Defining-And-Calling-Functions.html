Defining and Calling Functions

A function is a reusable block of code that performs a specific task. They are the backbone of organized, efficient programming because they allow you to write a piece of logic once and execute it many times from different parts of your program. Functions make code modular and easier to debug.

Concept: Functions are like recipes. The recipe itself is the definition, it describes the steps. Cooking the meal using the recipe is calling the function. You can call the same recipe (function) multiple times.

Part 1: Defining the Function

A function definition tells the computer what the function is named, what data it needs (parameters), and what code it should execute.

function greet(name) // function keyword, name, and parameter(s)
{
    console.log('Hello, ' + name + '!');
}

// The function is defined, but nothing has happened yet.

Key Terms:

  • function Keyword: Tells JavaScript you are creating a function.
  • Name (`greet`): How you will refer to and call the function later.
  • Parameter (`name`): A variable listed in the parentheses that holds the data required to run the function.

Part 2: Calling the Function

To make the function run its code, you must call it by its name, followed by parentheses containing the specific values (arguments) you want to pass in.

greet('Alice'); // Calling the function with the argument 'Alice'
greet('Bob');   // Calling it again with a different argument

// Outputs: 
// Hello, Alice!
// Hello, Bob!

Arguments vs. Parameters:

When you define the function, you list parameters (name). When you call the function, you provide arguments (Alice, Bob), which are the actual values the function uses.
