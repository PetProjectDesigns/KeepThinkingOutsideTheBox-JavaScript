Arrow Functions (`=>`)

Arrow functions provide a simpler and shorter syntax for defining functions, especially when they only perform a single action. They are defined using the fat arrow (=>) notation, making your code more readable and concise.

Standard Function: function add(a, b) { return a + b; }
Arrow Function: const add = (a, b) => a + b;

The Basic Multi-Line Arrow Function

For functions that have multiple statements (like logging to the console and returning a value, or using control flow), you must use curly braces {} and the explicit return keyword.

const greet = (name) => {
    
    console.log('Hello, ' + name + '!');
};

// Call:
greet('Arrow Fan'); // Outputs: Hello, Arrow Fan!

Single-Line Arrow Functions (Implicit Return)

If your function consists of a single `return` statement, you can omit both the curly braces {} and the `return` keyword. The value of the single expression is automatically returned.

// Single-line arrow function (no { } and no return keyword)
const add = (a, b) => a + b;

// result is 8
let result = add(5, 3);

// Outputs: 8
console.log(result);

The Explicit Return (Alternative Syntax)

Even if the arrow function is multi-line, if you only have a calculation, you can still use the standard multi-line syntax with the explicit return keyword for clarity.

const multiply = (a, b) => {
    
    return a * b;
};

// result is 24
let result = multiply(4, 6);

// Outputs: 24
console.log(result);

Arrow Function Rules

  • Conciseness: Shorter and cleaner than traditional functions.
  • Implicit Return: If no curly braces are used, the result of the single expression is automatically returned (e.g., (a, b) => a + b).
  • Parameters:
    • If there's only one parameter, the parentheses are optional: name => { ... }
    • If there are zero or multiple parameters, parentheses are required: () => { ... } or (a, b) => { ... }
