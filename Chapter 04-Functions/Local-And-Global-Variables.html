Scope, Lifetime, Local, and Global Variables

Scope defines where a variable is accessible within your code. Lifetime defines how long a variable exists in memory. Understanding these two concepts is crucial for avoiding unexpected errors and writing clean, reliable code.

Analogy: Think of Global scope as the company headquarters (accessible by everyone) and Local scope as a private office (accessible only by the person inside).

The Example Code: Demonstrating Scope

This code illustrates the boundary between variables declared outside a function (global) and those declared inside (local).

let globalVar = 'I am global'; // Global Scope: Defined outside the function.

function showVariables() {
    
    let localVar = 'I am local'; // Local Scope: Defined inside the function.
    
    console.log(globalVar); // 1. Accessible: Can access global from local scope.
    console.log(localVar);  // 2. Accessible: Can access local from local scope.
}

showVariables();

console.log(globalVar); // 3. Accessible: Can access global from global scope.

console.log(localVar); // 4. ERROR: localVar is only defined inside the function!

Global Variables

  • Definition: Variables declared outside any function or block.
  • Scope: Accessible from anywhere in your JavaScript program, including inside all functions.
  • Lifetime: They exist as long as the program is running.

Local Variables

  • Definition: Variables declared inside a function or a specific code block (using `let` or `const`).
  • Scope: Accessible only within the function or block where they are defined.
  • Lifetime: They are created when the function is called and destroyed (cease to exist) when the function finishes executing.
